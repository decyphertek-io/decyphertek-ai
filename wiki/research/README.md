# research/

**Broader findings, external references, brainstorming.**

## Filename convention
`<topic>.md`

Examples:
- `langchain-tool-calling.md`
- `chromadb-vs-qdrant.md`
- `mcp-protocol-notes.md`

## What goes here
- External links + brief summaries
- Comparisons (tool A vs tool B)
- Design brainstorming
- Spec excerpts / protocol notes

## What does NOT go here
- Conversation summaries (→ `memory/`)
- Working commands / fixes (→ `projects/`)

## Rules
- Always cite the source URL
- Keep notes concise — bullets and short paragraphs
- One topic per file

## Template

```markdown
# <Topic>

## Sources
- <https://example.com/article>

## Key points
- ...

## Relevance to Decyphertek.ai
- ...
```
